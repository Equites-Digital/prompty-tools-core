# Documentation site

Static site for `@prompty-tools/core`. Pages are plain HTML/CSS/JS - no
build step required to author content.

## Local preview

```sh
npm run docs:serve
```

This builds the package, copies `dist/index.js` into `docs/assets/lib/`, and
starts a local HTTP server at <http://localhost:4173>.

## Deployment

A GitHub Actions workflow (`.github/workflows/docs.yml`) builds and deploys
this folder to GitHub Pages on every push to `main`.

In the repository settings, set **Pages → Source** to **GitHub Actions**
(not "Deploy from a branch") so the workflow's artifact is what gets
served.

## Layout

| Path                  | Purpose                                       |
| --------------------- | --------------------------------------------- |
| `index.html`          | Landing page                                  |
| `getting-started.html`| Install, authenticate, first call             |
| `concepts.html`       | Pagination, errors, hooks, retries, abort     |
| `api/*.html`          | Reference per resource (one page each)        |
| `assets/styles.css`   | Shared stylesheet                             |
| `assets/runner.js`    | API key persistence + per-example execution   |
| `assets/lib/`         | Generated by `npm run docs:build` - `.gitignore`'d |

## Notes

- Every example runs in the browser via the actual published ESM bundle.
  The user's API key is stored in `localStorage` only.
- Examples are wrapped in an `AsyncFunction` that receives `client` and the
  `Prompty*Error` classes pre-bound, so example code never has to write
  `import` statements.
- Examples that mutate state (`create`, `update`, `delete`, `vote`,
  `setVisibility`, etc.) wear a `.danger` class - the runner shows a
  confirmation dialog before executing them.
- This folder is excluded from the npm publish (see `files` in
  `package.json` and `.npmignore`).
